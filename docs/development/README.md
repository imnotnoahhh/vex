# vex Development Docs

This directory contains maintainer-facing documentation for the vex repository.

## Included Docs

- [architecture.md](architecture.md) - Core architecture, data flow, filesystem layout, templates, and safe team-config flow
- [testing.md](testing.md) - Test strategy, failure-recovery coverage, CLI integration, and validation layers
- [releasing.md](releasing.md) - Versioning, release checklist, rollback process, and action/release automation

## When To Use These Docs

- Read [architecture.md](architecture.md) when changing core behavior, filesystem layout, or shell integration.
- Read [testing.md](testing.md) before adding new features, changing failure recovery, or updating CI checks.
- Read [releasing.md](releasing.md) when preparing a tagged release, updating the GitHub Action, or changing release automation.
