# vex Development Docs

This directory contains maintainer-facing documentation for the vex repository.

## Included Docs

- [architecture.md](architecture.md) - Core architecture, data flow, and filesystem layout
- [testing.md](testing.md) - Test strategy, validation layers, and coverage guidance
- [releasing.md](releasing.md) - Versioning, release checklist, and rollback process

## When To Use These Docs

- Read [architecture.md](architecture.md) when changing core behavior, filesystem layout, or shell integration.
- Read [testing.md](testing.md) before adding new features, changing validation strategy, or updating CI checks.
- Read [releasing.md](releasing.md) when preparing a tagged release or a hotfix.
