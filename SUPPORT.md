# Support

Thanks for using vex. This project is maintained in the open through GitHub issues and pull requests.

## Where To Get Help

- Read the user docs: [docs/README.md](docs/README.md)
- Check the command reference: [docs/guides/command-reference.md](docs/guides/command-reference.md)
- Run local diagnostics:

```bash
vex doctor --verbose
vex globals --verbose
```

## Reporting Bugs

Open a bug report at [GitHub Issues](https://github.com/imnotnoahhh/vex/issues/new/choose) and include:

- `vex --version`
- macOS version and CPU architecture
- shell name and version
- the exact command sequence that reproduces the issue
- relevant `vex doctor --json` output, with private paths or tokens removed

## Feature Requests

Use the feature request template and describe:

- the workflow you are trying to support
- what you tried with current vex commands
- why the existing behavior is insufficient

## Security Issues

Do not open a public issue for vulnerabilities. Follow [SECURITY.md](SECURITY.md).
